PZ Modsmith
===========

PZ Modsmith is a local helper for building and checking Project Zomboid
dedicated-server mod lists.

Give it either a Project Zomboid console log from a server join attempt or a
plain list of Steam Workshop IDs/URLs. It scans your local Steam Workshop
downloads, reads each mod.info file, and produces draft server config lines:

  WorkshopItems=...
  Mods=...

It is especially useful for Workshop items that contain more than one Mod ID,
old Build 41 variants, single-player variants, compatibility patches, libraries,
or optional add-ons.


Requirements
============

CLI mode:

  Python 3.10 or newer should be enough. The CLI uses only the Python standard
  library.

Web UI mode:

  Flask is required.

  python3 -m venv .venv
  source .venv/bin/activate
  pip install flask


Running It
==========

From this directory, you can run either entry point:

  python3 pz_modsmith.py --help

or:

  python3 -m pz_modsmith --help


Web UI
======

Start the local browser UI:

  python3 pz_modsmith.py --web

By default it binds to:

  http://127.0.0.1:8765

Optional host and port:

  python3 pz_modsmith.py --web --host 127.0.0.1 --port 9000

In the UI, provide:

  1. Your Steam Workshop content path for Project Zomboid.
  2. A console.txt upload, pasted console text, Workshop IDs, or Steam URLs.
  3. Review selections for multi-Mod-ID Workshop items.
  4. Copy the generated WorkshopItems= and Mods= lines.

The web UI can also download a ZIP containing the same reports generated by CLI
mode.


CLI Examples
============

Analyze a console log from a failed or successful server join attempt:

  python3 pz_modsmith.py --console ~/Zomboid/console.txt

Analyze a text file containing Workshop IDs or Steam Workshop URLs:

  python3 pz_modsmith.py --ids-file ./workshop-ids.txt

Write reports to a custom output directory:

  python3 pz_modsmith.py --console ~/Zomboid/console.txt --out ./reports

Use a custom Steam Workshop path:

  python3 pz_modsmith.py \
    --ids-file ./workshop-ids.txt \
    --workshop-path "/path/to/SteamLibrary/steamapps/workshop/content/108600"


Workshop Path
=============

PZ Modsmith expects the Project Zomboid Workshop content folder, usually named
108600. It checks these common Linux paths by default:

  /mnt/storage/SteamLibrary/steamapps/workshop/content/108600
  ~/.steam/debian-installation/steamapps/workshop/content/108600
  ~/.steam/steam/steamapps/workshop/content/108600
  ~/.local/share/Steam/steamapps/workshop/content/108600

If your Steam library lives somewhere else, pass --workshop-path or enter the
path in the web UI.


Input Formats
=============

Console log input:

  Use ~/Zomboid/console.txt from a server join attempt. PZ Modsmith extracts
  Workshop IDs from ID=... log lines and also looks for active Mod IDs that the
  game actually loaded. Those active IDs are used as high-confidence selections
  when a Workshop item contains multiple variants.

IDs file or pasted free text:

  The input can contain plain numeric Workshop IDs or Steam Workshop URLs such
  as:

    https://steamcommunity.com/sharedfiles/filedetails/?id=1234567890


Generated Reports
=================

CLI mode writes reports to ./pzmodsmith-output by default:

  summary.txt
    Counts of requested Workshop IDs, single-ID items, multi-ID items, missing
    items, diagnostics, and the Workshop path used.

  draft_server_lines.ini
    Draft WorkshopItems= and Mods= lines for your server config.

  workshop_ids.txt
    The normalized Workshop ID list, one ID per line.

  single_mod_items.txt
    Workshop items where only one Mod ID was found.

  review_multi_mod_items.txt
    Workshop items with multiple Mod IDs and the selected best guess.

  warnings_bad_smells.txt
    Items with warning patterns such as B41, single-player, legacy, no-MP,
    optional, patch, compatibility, addon, deprecated, or do-not-activate text.

  missing_workshop_items.txt
    Workshop IDs that were not found locally or had no readable mod.info.

  connection_failure_diagnosis.txt
    Only generated when console-log diagnostics are found.

  result.json
    Structured analysis data for tooling or debugging.


How Selection Works
===================

For each Workshop ID, PZ Modsmith searches recursively for mod.info files.

If one unique Mod ID is found, it selects that Mod ID automatically.

If multiple Mod IDs are found, it ranks them with simple text heuristics:

  Positive hints:
    B42, Build 42, MP, multiplayer, current, fixed, maintained

  Negative hints:
    B41, Build 41, SP, single-player, legacy, pre-42, pre-MP, disable,
    do-not-activate, optional, patch, compatibility, addon, no-MP, unstable,
    deprecated

If the console log shows that a Mod ID actually loaded, that Mod ID receives a
large confidence boost and is selected unless there is conflicting evidence.

The tool also flags possible dependencies and support mods, including library,
framework, API, utility, tile-pack, vehicle-framework, patch/addon, and
mod.info require/required entries.


Connection Diagnostics
======================

When given console text, PZ Modsmith looks for common Project Zomboid connection
failure patterns, including:

  WorldDictionary fatal load errors
  Missing dictionary strings on the client
  Missing client scripts for dictionary info
  Lua runtime errors in mods
  Severe Lua file errors under Workshop item folders

For dictionary and missing-script findings, it also scans downloaded Workshop
text files for the missing key and reports whether the key appears in selected
or unselected Workshop items.

Diagnostics are useful hints, not proof. A stale world save, bad load order,
wrong variant, missing dependency, or outdated Workshop download can produce
similar symptoms.


Important Notes
===============

Review the generated Mods= line before using it on a real server. Some Workshop
items intentionally include several Mod IDs, patches, compatibility modules, or
optional add-ons. PZ Modsmith chooses a best guess, but it cannot know your exact
server design.

The tool reads local Workshop downloads only. It does not download mods from
Steam.

If a Workshop ID is listed as missing, make sure the item is subscribed,
downloaded, and present under the 108600 Workshop content folder.


Project Layout
==============

  pz_modsmith.py
    Compatibility launcher script.

  pz_modsmith/cli.py
    Command-line argument parsing and top-level CLI flow.

  pz_modsmith/web.py
    Flask web UI and report download routes.

  pz_modsmith/scanner.py
    mod.info scanning, scoring, and warning flag detection.

  pz_modsmith/log_parser.py
    Workshop ID and active Mod ID extraction from logs or free text.

  pz_modsmith/analysis.py
    Core analysis pipeline and selection application.

  pz_modsmith/diagnostics.py
    Console-log failure pattern extraction and local file enrichment.

  pz_modsmith/reports.py
    Text, JSON, INI, and ZIP report generation.

  pz_modsmith/models.py
    Dataclasses for analysis results, Workshop items, mods, and findings.
